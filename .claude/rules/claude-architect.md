# Claude Architect Study Content Rules

When editing or generating content for the Claude Architect Study Guide:

## 1. Provenance and Attribution
- Every claim about Claude API behavior or SDK features must be attributable to official Anthropic documentation (as summarized in the guide).
- Use clear headings and subheadings to maintain hierarchy.

## 2. Formatting Standards
- Use Markdown tables for comparative data (e.g., API fields, tool choice parameters).
- Use code blocks with appropriate language tags for JSON request/response examples.
- Include "Critical" or "Exam Trap" callouts using blockquotes.

## 3. Impeccable Design cues
- **Vertical Rhythm:** Ensure consistent spacing between chapters and sections.
- **Scannability:** Use bold text for key terms and identifiers.
- **No SaaS Slop:** Avoid generic marketing language. Focus on engineering precision.

## 4. Verification
- After updating content, run an internal "audit" to ensure no contradictions were introduced.
